My Applications

NightOut

Click the image to see a live demo!

My Role: Web Developer

App Description: An app implementing OOP and two different APIs to look up events and breweries by city and state

Source Code: View on GitHub

Project Info: The idea behind the app was to provide a way to look up local events and breweries to help the user plan a night out where ever they plan on going. This was done by making AJAX requests to a couple of public APIs based on a city and state provided by the user.

This was my first time using APIs, so it was a challenge finding out how to use the information that I received. For example, I wanted to check if there was no event data retrieved by the AJAX request, so initially I thought to check if the request was a falsy value. However, the problem with this approach was that I never received a falsy value from the request! So instead I had to devise a way to check if specifically the piece of data that I requested (the array of events) was empty.

There is a lot that I want to add to this project! Features such as filtering events and breweries by type and showing the location of the events and nearby breweries are the next challenges I am looking to tackle.

Wicked Sales

Click the image to see a live demo!

My Role: Full Stack Developer

App Description: An app utilizing React.js and Node.js for witches looking to buy witching products

Source Code: View on GitHub

Project Info: The goal going into this project was to create an app that allowed it's users to view products from a catalog, click on them to see more details, add those products to a cart, view the cart, and then checkout the cart by entering in their information. I had a lot of fun designing the site around a occult theme and working through the challenge of building and designing a back end for the first time.

I came to the realization during this project that designing well-thought-out Express.js endpoints can save a lot of time validating information in client side requests. The server's job is to serve information, but a poorly formed request can create confusion and make it harder to debug. Accounting for those problems in the endpoints and sending relavent status codes and error messages will reduce time spent searching for problems down the line.

With this in mind, there are a lot of checks I would like to add to the checkout form! Namely, I would like to ensure that the credit card and address information input by the user is checked for correct formatting before inserting the information to the table.
